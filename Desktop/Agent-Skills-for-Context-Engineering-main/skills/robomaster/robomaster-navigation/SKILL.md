# RoboMaster 导航系统

## 系统架构

导航系统采用四层架构设计：

### 感知层
- **激光雷达**：Livox Mid360 等
- **IMU**：惯性测量单元
- **视觉相机**：辅助定位
- **里程计**：轮式编码器

功能：传感器融合，扩展卡尔曼滤波，状态估计

### 控制层
- **底盘运动学解算**
- **速度跟踪控制**
- **姿态稳定控制**
- **功率限制控制**

### 规划层
- **全局路径规划**：A*、Dijkstra 算法
- **局部避障**：DWA、TEB 算法
- **动态路径重规划**

### 决策层
- **有限状态机管理**
- **行为决策**
- **任务调度**
- **威胁评估**

## SLAM 建图与定位

### SLAM 方案选择

| SLAM 类型 | 优点 | 缺点 | 适用场景 |
|---------|------|------|---------|
| LiDAR SLAM | 精度高，鲁棒性强 | 成本高 | 室内/室外 |
| Visual SLAM | 成本低，信息丰富 | 受光照影响 | 纹理丰富环境 |
| LiDAR-IMU 融合 | 精度高，抗干扰 | 系统复杂 | 高精度需求 |

### 常用 SLAM 算法
- Cartographer：Google 开源，建图效果好
- GMapping：经典 2D SLAM 算法
- LOAM/LIO-SAM：LiDAR-IMU 融合方案

### 核心模块
1. **激光特征提取**：边缘特征、平面特征
2. **特征匹配**：帧间匹配、局部地图匹配
3. **位姿优化**：非线性最小二乘优化
4. **滑动窗口边缘化**：限制计算量

## 路径规划

### 全局路径规划
- **算法**：A*、Dijkstra、RRT*
- **输入**：起点、终点、地图
- **输出**：全局路径点序列

### 局部路径规划
- **算法**：DWA（动态窗口法）、TEB（时间弹性带）
- **输入**：全局路径、实时障碍物
- **输出**：速度指令（vx、vy、vw）

### DWA 算法原理
1. 生成速度采样空间
2. 对每个速度预测轨迹
3. 轨迹评价（距离目标、速度、障碍物）
4. 选择最优速度指令

## 决策系统

### 有限状态机（FSM）

典型状态：
- **STATE_IDLE**：空闲待命
- **STATE_PATROL**：巡逻状态
- **STATE_CHASE**：追击敌人
- **STATE_DEFEND**：防守状态
- **STATE_ESCAPE**：紧急撤退
- **STATE_RECHARGE**：回充状态

### 状态转换条件
- 检测到敌人 → 追击
- 敌人接近 → 防守
- 血量低 → 撤退/回充
- 丢失敌人 → 巡逻
