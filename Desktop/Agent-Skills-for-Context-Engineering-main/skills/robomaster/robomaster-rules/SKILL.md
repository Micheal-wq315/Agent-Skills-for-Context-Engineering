# RoboMaster 裁判系统规则讲解

## 比赛概述

RoboMaster 机甲大师超级对抗赛是由大疆创新（DJI）主办的大学生机器人竞技比赛。对战双方需自主研发不同种类和功能的机器人，在指定的比赛场地内进行战术对抗，通过操控机器人发射弹丸攻击对方机器人和基地。比赛结束时，基地剩余血量高的一方获得比赛胜利。

比赛采用类似王者荣耀的对抗模式，集技术性、对抗性、观赏性于一体，对参赛队员的机械设计、电子开发、嵌入式编程、机器视觉算法及团队协作能力都提出了极高要求。

## 一、机器人类型详解

### （一）步兵机器人

步兵是赛场上数量最多、机动性最强的作战单位，每队配置 2-3 台。

| 属性 | 规格 |
|------|------|
| **定位** | 灵活机动的输出单位 |
| **弹丸** | 17mm 标准弹丸 |
| **底盘** | 麦克纳姆轮 / 全向轮底板 |
| **云台** | GM6020 高速电机驱动 |
| **电控架构** | 双板控制（A 型底盘板 + C 型云台板） |
| **血量** | 中等水平 |
| **数量优势** | 骚扰、支援、侦查多重角色 |

**关键战术**：
- **小陀螺战术**：底盘持续旋转配合云台稳定瞄准，使机器人以旋转姿态快速移动的同时保持持续火力输出
- **全向移动**：麦轮实现任意方向平移运动

**双板控制架构通信方式**：
- CAN 总线：底盘状态、云台目标
- 串口 UART：低延迟数据同步
- 底盘板 → 云台板：底盘角度、IMU 姿态
- 云台板 → 底盘板：目标方位、射击指令

### （二）英雄机器人

作为主力输出火力平台，英雄发射 42mm 大弹丸：

| 属性 | 规格 |
|------|------|
| **定位** | 主力输出火力平台 |
| **弹丸** | 42mm 大弹丸，单发伤害高 |
| **底盘** | 大型底盘 + 多 M3508 驱动 |
| **云台** | GM6020 稳定瞄准 |
| **血量** | 较高 |
| **劣势** | 体积大，机动性较低 |

英雄战术要义：利用高血量承担正面战斗压力，配合步兵完成火力压制。由于机动性差，需要工程机器人辅助获取增益。

### （三）工程机器人

| 属性 | 规格 |
|------|------|
| **定位** | 辅助支援单位 |
| **攻击能力** | 不配备 |
| **核心功能** | 兑换金币、救援阵亡友军、触发场地机关 |
| **底盘** | 中型底盘 + M3508 |
| **执行机构** | 机械臂/抓取机构 |

工程核心策略：
- 开局抢金币 → 兑换增益 → 获取装备优势
- 救援阵亡友军 → 减少战场损耗
- 触发机关 → 改变战局

### （四）空中机器人

| 属性 | 规格 |
|------|------|
| **平台** | 四旋翼飞行平台 |
| **载荷** | 轻量级发射机构 |
| **功能** | 空中观测、战术信息传递、特定情况空中打击 |

空中机器人的制空权对于战场信息获取至关重要，是第一手情报来源。

### （五）哨兵机器人

| 属性 | 规格 |
|------|------|
| **定位** | 基地自动防御单位 |
| **运行模式** | 完全自主运行，无手动操控 |
| **移动范围** | 基地轨道（固定路径） |
| **核心功能** | 自动识别+攻击进入防御范围的敌方机器人 |
| **必要能力** | SLAM 自主定位、路径规划、目标识别 |

### （六）雷达机器人

| 属性 | 规格 |
|------|------|
| **定位** | 战场态势感知 |
| **部署位置** | 己方半场 |
| **核心功能** | 探测敌方机器人位置、运动状态 |
| **输出信息** | 敌方坐标、威胁等级、运动轨迹 |

雷达为全体操作员提供全局战场信息，是战术决策的依据。

## 二、比赛流程与胜负判定

### （一）比赛时间结构（7 分钟赛制）

| 阶段 | 说明 |
|------|------|
| **准备阶段** | 双方进行机器人部署和系统检查 |
| **启动阶段** | 机器人激活，状态确认 |
| **战术阶段** | 主战斗时间，抢夺机关和击杀 |
| **基地摧毁阶段** | 决胜时刻 |

### （二）胜负判定机制

- **核心依据**：比赛结束时基地剩余血量
- **获胜条件**：基地血量高的一方获胜
- **提前获胜**：一方基地血量归零，另一方立即获胜

### （三）血量系统详解

每个机器人拥有独立血量值，受弹丸攻击时根据以下因素扣血：

| 因素 | 影响 |
|------|------|
| **弹丸类型** | 17mm 标准弹 vs 42mm 大弹丸 |
| **命中部位** | 不同部位扣血不同 |

血量归零 → 机器人进入「死亡」状态 → 失去作战功能 → 需工程机器人救援复活

## 三、场地机关与增益系统

### （一）地形要素

| 元素 | 影响 |
|------|------|
| **标准地面** | 正常通行 |
| **飞坡** | 需要动力跨越 |
| **资源岛** | 获取资源 |
| **高地** | 视野优势、战术高点 |
| **基地** | 双方核心保护目标 |

### （二）机关类型与功能

| 机关 | 功能 |
|------|------|
| **增益机关** | 提供攻击/防御增益 |
| **能量机关** | 激活特殊能力 |
| **兑换机关** | 金币兑换装备 |
| **救援点** | 救援阵亡友军 |

## 五、犯规类型与处罚

| 类型 | 说明 | 处罚等级 |
|------|------|---------|
| **机器人尺寸超标** | 静态/动态尺寸超过规定 | 警告/罚下 |
| **超功率运行** | 底盘/发射功率超过限制 | 扣血/警告 |
| **违规接触** | 非规则允许的物理接触 | 警告/扣血 |
| **违规弹丸** | 使用非标准弹丸 | 取消资格 |
| **操作员违规** | 操作区违规行为 | 警告/罚下 |

## 六、视觉标签系统

比赛场地设有视觉标签用于机器人定位和识别：

| 标签类型 | 用途 |
|---------|------|
| **数字标签** | 机器人身份识别 |
| **能量机关标签** | 能量机关识别与瞄准 |
| **基地标签** | 基地位置定位 |

标签识别是视觉系统的核心任务之一。
