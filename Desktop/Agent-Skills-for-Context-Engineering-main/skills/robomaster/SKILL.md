# 伪代码：混合检索流程
def hybrid_retrieve(query, top_k=5):
    # 1. 向量检索（语义匹配）
    query_embedding = embed_model.encode(query)
    vector_results = vector_db.search(query_embedding, k=top_k*2)
    
    # 2. 关键词检索（BM25）
    keyword_results = bm25_index.search(query, k=top_k*2)
    
    # 3. RRF融合排序
    fused_results = reciprocal_rank_fusion(
        vector_results, 
        keyword_results,
        k=60  # RRF参数
    )
    
    return fused_results[:top_k]
```

## 六、交互层功能

### 6.1 代码解释器

用户可以上传代码片段，技能进行分析：

**示例对话**：

> **用户**：请分析这段PID代码的稳定性
> ```c
> pid_output = kp*error + ki*integral + kd*derivative;
> ```
> 
> **助手**：
> 这段代码存在以下问题：
> 1. **积分饱和**：未对integral进行限幅，可能导致积分饱和
> 2. **微分冲击**：derivative直接计算，噪声敏感
> 
> 改进建议：
> ```c
> // 积分限幅
> integral = clamp(integral + error, -INTEGRAL_MAX, INTEGRAL_MAX);
> 
> // 微分滤波
> derivative = alpha * (error - last_error) + (1-alpha) * last_derivative;
> 
> pid_output = kp*error + ki*integral + kd*derivative;
> ```

### 6.3 参数推荐

> **用户**：我的云台电机是GM6020，负载0.5kg，该用什么PID参数？
> 
> **助手**：
> 根据GM6020规格和0.5kg负载，推荐初始PID参数：
> 
> | 控制环 | Kp | Ki | Kd | 说明 |
> |--------|----|----|----|------|
> | 角度环 | 80 | 2 | 20 | 快速响应 |
> | 速度环 | 15 | 0.5 | 0 | 平滑跟踪 |
> 
> 调参步骤：
> 1. 先调Kp：从小到大，直到出现轻微振荡
> 2. 加入Ki：消除稳态误差
> 3. 加入Kd：抑制超调

### 6.4 故障排查向导

多轮对话引导定位问题：

> **用户**：我的电机不转
> 
> **助手**：请检查以下几点：
> 1. **CAN指示灯**是否闪烁？（是/否）
> 
> **用户**：否
> 
> **助手**：CAN通信可能有问题。请检查：
> - CAN_H和CAN_L接线是否正确
> - 终端电阻120Ω是否接入
> - 波特率是否设置为1Mbps
> 
> 2. **电机是否发热**？（是/否）
> 
> **用户**：是
> 
> **助手**：电机可能堵转或电流过大。请检查：
> - 机械结构是否卡死
> - 电流限制是否设置正确（M3508建议<10A）
> - 编码器反馈是否正常

> // 输入：vx, vy, vw（期望速度）

> void mecanum_calc(float vx, float vy, float vw, float* wheel_speed) {

>     float B = vy;

>     wheel_speed[2] = (A - B - C) / WHEEL_RADIUS;  // 左后
>     wheel_speed[3] = (A + B - C) / WHEEL_RADIUS;  // 右后
