# RoboMaster 通讯协议

## CAN 总线通信

### CAN 总线特点
- **多主控制模式**：多节点可同时发送
- **优先级仲裁机制**：高优先级消息优先传输
- **完善错误检测机制**：CRC 校验、位错误、填充错误
- **高速传输能力**：最高 1Mbps

### C 型板 CAN 接口配置
- **CAN1 引脚**：PB8 (TX)、PB9 (RX) 或 PA12 (TX)、PA11 (RX)
- **CAN2 引脚**：PB12 (TX)、PB13 (RX) 或 PB5 (TX)、PB6 (RX)
- **终端电阻**：需要 120Ω 匹配
- **波特率**：通常设置为 1Mbps

### CAN 数据帧格式

#### 发送数据帧（主控 → 电调）
标准 8 字节格式：
- **Byte 0-1**：控制值低字节、高字节（电流指令，int16）
- **Byte 2-7**：保留，通常填充 0

#### 接收数据帧（电调 → 主控）
标准 8 字节格式：
- **Byte 0-1**：转子角度（0-8191），12 位分辨率
- **Byte 2-3**：转子转速（有符号 int16）
- **Byte 4-5**：实际电流（有符号 int16）
- **Byte 6-7**：温度（摄氏度）

## 串口通信

### 双板通信（底盘板 vs 云台板）
- **协议选择**：UART 串口或 CAN 总线
- **数据内容**：
  - 底盘 → 云台：底盘状态、IMU 数据
  - 云台 → 底盘：控制指令、目标状态
- **传输频率**：100Hz - 500Hz

### 视觉通信
- **协议选择**：UART 串口、USB 或 Ethernet
- **数据内容**：
  - 视觉 → 电控：目标坐标、距离、置信度
  - 电控 → 视觉：机器人状态、控制模式
- **传输频率**：30Hz - 60Hz

## 裁判系统接口

### 裁判系统数据内容
- **机器人状态**：血量、等级、位置
- **比赛状态**：时间、阶段、比分
- **场地信息**：增益机关状态、补给站信息
- **机器人 ID**：红蓝方身份识别

### 通信协议
- **接口**：USART 串口
- **数据格式**：自定义二进制协议
- **接收频率**：10Hz - 100Hz
