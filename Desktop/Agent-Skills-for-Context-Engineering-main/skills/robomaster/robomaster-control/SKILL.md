# RoboMaster 电控系统

## 系统架构

电控系统采用分层架构设计：

### 硬件抽象层 (HAL)
封装底层硬件操作，包括：
- GPIO 控制
- 定时器配置
- ADC 采样
- CAN 通信

### 驱动层
针对特定外设的驱动程序：
- 电机驱动
- 传感器驱动
- 通信协议栈

### 中间件层
- 实时操作系统 (FreeRTOS)
- 通信协议栈
- 数据处理算法

### 应用层
实现具体功能逻辑：
- 底盘运动控制
- 云台姿态控制
- 发射机构控制
- 裁判系统交互

## 双板控制架构

步兵机器人采用双板控制：
- **底盘控制板**：底盘运动学解算、功率限制、电机驱动
- **云台控制板**：云台姿态稳定、目标跟踪、射击控制
- **通信方式**：CAN 总线或串口

优势：
- 职责明确，便于调试和维护
- 控制周期可独立优化（底盘 2ms，云台 1ms）
- 故障隔离好，单板故障不影响其他功能

## PID 控制算法

### PID 基本原理

PID 控制器由三个环节组成：
- **P（比例）**：快速响应误差
- **I（积分）**：消除稳态误差
- **D（微分）**：抑制超调和振荡

### 串级 PID 控制架构

RoboMaster 电机控制采用三环 PID：
- **电流环（最内环）**：直接控制电机电流，实现力矩输出，响应最快
- **速度环（中环）**：以目标转速为输入，输出作为电流环给定值
- **角度环（最外环）**：根据目标角度计算所需转速，实现精确位置控制

### PID 参数整定流程

1. **整定 Kp**：从 0 开始逐渐增大直到出现振荡，然后适当减小
2. **整定 Ki**：引入积分作用消除稳态误差，注意防止积分饱和
3. **整定 Kd**：增加微分作用抑制超调和振荡
4. **微调优化**：整体优化调整

### 典型参数范围

| 控制对象 | Kp 范围 | Ki 范围 | Kd 范围 |
|---------|--------|--------|--------|
| 底盘速度环 | 10-50 | 0.1-1 | 0-10 |
| 云台角度环 | 50-200 | 0.5-5 | 10-50 |
| 发射机构 | 5-20 | 0.1-0.5 | 0-5 |
| 拨弹电机 | 15-30 | 0.2-1 | 0-5 |

## 底盘运动控制

### 麦克纳姆轮运动学

麦轮底盘通过组合 4 个轮子的力合成任意方向的运动。

运动学解算公式：
```
wheel_speed[0] = vx + vy + vw * chassis_radius
wheel_speed[1] = vx - vy + vw * chassis_radius
wheel_speed[2] = vx - vy - vw * chassis_radius
wheel_speed[3] = vx + vy - vw * chassis_radius
```

### 功率限制

赛事对机器人功率有限制：
- 峰值功率限制
- 持续功率限制

需要实时监测电机电流和电压，计算瞬时功率，通过调整电机输出实现功率约束。

### 斜坡规划

实现平滑的速度变化，避免阶跃输入导致的机械冲击和打滑现象。

## 云台控制策略

### 6020 角度双环控制

由两个反馈环组成：
- **角度环（外环）**：控制电机最终位置，计算期望转速
- **速度环（内环）**：接收角度环输出作为目标转速，计算期望电流

### 小陀螺战术实现

核心思想：实现底盘旋转与云台稳定的解耦控制。

实现步骤：
1. 获取底盘与云台的相对角度
2. 建立坐标转换关系
3. 实现解耦控制

关键：理解云台坐标和底盘坐标的转换数学模型。
